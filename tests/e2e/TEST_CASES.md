# E2E Test Cases — Telegram ↔ MAX Bridge

## Prerequisites

1. Установить тестовые зависимости (один раз):
   ```bash
   pip install -r requirements-test.txt
   ```

2. Авторизовать TG-аккаунт для тестов (один раз):
   ```bash
   ./bridge.sh test-auth
   ```

3. Создать конфиг тестов (один раз):
   ```bash
   cp tests/e2e/e2e_config.example.yaml tests/e2e/e2e_config.yaml
   # заполнить: user_name, tg_chat_id, max_chat_id
   ```

4. Запустить бридж:
   ```bash
   ./bridge.sh start          # локально
   # или
   ./bridge.sh docker up      # в Docker
   ```

## Запуск тестов

```bash
# Все тесты
./bridge.sh test

# Конкретный кейс по ID (работает только для кейсов с реальными test-функциями)
./bridge.sh test -k T01
./bridge.sh test -k P01
./bridge.sh test -k M13
./bridge.sh test -k E05

# По направлению
./bridge.sh test -k "tg_to_max"
./bridge.sh test -k "max_to_tg"

# По маркеру (группе)
./bridge.sh test -m text
./bridge.sh test -m reply
./bridge.sh test -m edit
./bridge.sh test -m delete
./bridge.sh test -m echo
./bridge.sh test -m formatting
./bridge.sh test -m edge
./bridge.sh test -m media
./bridge.sh test -m reaction
```

## Status legend

- Авто: `🤖` = automated test (`test_T01_*`, `test_P01_*`, etc.), `👤` = manual only, `➖` = N/A или covered by другой тест (ID в скобках)
- Статус: `🔲` not tested, `✅` passed, `❌` failed, `⚠️` skipped, `➖` N/A
- Последний прогон: `—` initially, updated automatically by conftest after each pytest run

> **Note:** T03/T04/T05 (фото/видео TG→MAX) и M03/M04/M05 (фото/видео MAX→TG) не имеют
> отдельных тест-функций — сценарии полностью покрыты секцией **Медиа (P01–P15)**.
> Для запуска используй `./bridge.sh test -k P01` и т.д.
