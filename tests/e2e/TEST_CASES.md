# E2E Test Cases — Telegram ↔ MAX Bridge

## Prerequisites

1. Установить тестовые зависимости (один раз):
   ```bash
   pip install -r requirements-test.txt
   ```

2. Авторизовать TG-аккаунты для тестов (один раз):
   ```bash
   ./bridge.sh test-auth              # primary user
   ./bridge.sh test-auth --user mary  # second user (для двухпользовательских тестов)
   ```

3. Создать конфиг тестов (один раз):
   ```bash
   cp tests/e2e/e2e_config.example.yaml tests/e2e/e2e_config.yaml
   # заполнить: user_name, second_user_name, tg_chat_id, max_chat_id
   ```

4. Запустить бридж:
   ```bash
   ./bridge.sh start          # локально
   # или
   ./bridge.sh docker up      # в Docker
   ```

## Запуск тестов

```bash
# Все тесты
./bridge.sh test

# Конкретный кейс по ID
./bridge.sh test -k T01
./bridge.sh test -k P01

# По направлению
./bridge.sh test -k "tg_to_max"
./bridge.sh test -k "max_to_tg"

# По маркеру (группе)
./bridge.sh test -m text
./bridge.sh test -m formatting
./bridge.sh test -m media
./bridge.sh test -m reaction
./bridge.sh test -m twouser
./bridge.sh test -m edge
```

## Status legend

- Авто: `🤖` = automated test, `👤` = manual only, `➖` = N/A или covered by другой тест
- Статус: `✅` passed, `❌` failed, `⚠️` flaky (rate limits), `⏭️` skipped (ограничение протокола), `🔲` not tested
- Последний прогон обновляется автоматически conftest после каждого запуска pytest

> **Note:** T03/T04/T05 и M03/M04/M05 не имеют отдельных тест-функций — покрыты секцией **Медиа (P01–P15)**.
